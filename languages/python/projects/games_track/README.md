# Python Games Track: From Logic to Graphics

Welcome to the Games Track. Here you will learn how to transform logical thinking into interactive experiences, starting from the command line and moving into 2D graphical environments.

## Prerequisites
Before starting this track, you should be comfortable with:
- **Core Logic:** Loops, conditional statements, and boolean logic.
- **Data Structures:** Lists and dictionaries for managing game state.
- **Functions:** Modularizing game logic (e.g., `check_win()`, `update_player()`).
- **OOP (Object-Oriented Programming):** Understanding classes and objects is crucial for Pygame (Sprites, Game Engines).

## Core Libraries
- `random`: For generating unpredictable game elements.
- `colorama`: For enhanced CLI visuals.
- `pygame`: The industry standard for 2D games in Python.
- `arcade`: A modern alternative to Pygame with a focus on simplicity and modern Python features.

## Projects Roadmap

### Project 1: Terminal RPG / Adventure (Logic & CLI)
Build a text-based adventure game where the player navigates through rooms, manages an inventory, and fights enemies using turn-based logic.
- **Focus:** State management, input validation, and complex branching logic.

### Project 2: Classic Arcade Clone (Pygame Basics)
Recreate a classic like *Pong*, *Snake*, or *Space Invaders*.
- **Focus:** The Game Loop (Initialize -> Handle Events -> Update -> Draw), collision detection, and basic 2D physics.

### Project 3: 2D Platformer or Top-down Shooter (Advanced Graphics)
Develop a more complex game with multiple levels, scrolling backgrounds, animations, and sound effects.
- **Focus:** Sprite groups, asset management, advanced collision, and game AI (enemy behavior).
