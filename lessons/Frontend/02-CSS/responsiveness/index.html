שימוש במדיה קווריז (Using Media Queries)

מדיה קווריז (Media Queries) הם תכונה של CSS המאפשרת להתאים את תוכן הדף למאפייני המכשיר. זהו הבסיס לעיצוב רספונסיבי.

Mobile First: אסטרטגיה מומלצת שבה מעצבים קודם כל עבור מסכים קטנים ומוסיפים שיפורים למסכים גדולים יותר באמצעות min-width.

למה להשתמש ב-rem? (px vs rem)

השימוש ב-rem מאפשר ליצור עיצוב יחסי. המטרה היא שכל ה-UI (טקסט, ריווחים, ואפילו תיבות) "ינשום" ויגדל יחד עם העדפות המשתמש.

שימו לב להבדל:
  • Browser Zoom (Ctrl +): מגדיל את כל המסך (גם פיקסלים וגם rem). במצב זה לא תראו הבדל בין התיבות.
  • Browser Font Settings: הגדרה עמוקה בדפדפן (למשל בכרום: Settings -> Appearance -> Font size). הגדרה זו משנה רק יחידות יחסיות כמו rem.
הניסוי המנצח: שנו את גודל הפונט בהגדרות הדפדפן ל-"Very Large". התיבה הירוקה (ב-rem) תגדל ותהיה רחבה וגדולה יותר, בעוד התיבה האדומה (ב-px) תישאר קטנה והטקסט בתוכה עלול "להישבר" או לצאת החוצה.

עיצוב קבוע (Fixed)

אני תקוע ב-250px

אם המשתמש יגדיל פונט, התיבה הזו תישאר קטנה והטקסט יצא ממנה.

עיצוב יחסי (Scalable)

אני גדל עם המשתמש (rem)

התיבה הזו תגדל יחד עם הטקסט ותשמור על המראה שלה.

יחידות מידה יחסיות (em vs rem)

במקום להשתמש בפיקסלים (px) שהם יחידה קבועה, מומלץ להשתמש ביחידות יחסיות שמאפשרות לאתר "לנשום" ולהתאים את עצמו להגדרות המשתמש.

1. rem (Root EM)

יחידה היחסית לאלמנט השורש (ה-html). ברוב הדפדפנים, ברירת המחדל היא 1rem = 16px.

  • יתרון: עקביות. 1rem תמיד יהיה באותו גודל בכל מקום באתר.
  • שימוש נפוץ: גדלי פונטים, ריווחים (padding/margin) ופריסה כללית.

2. em

יחידה היחסית לגודל הפונט של האלמנט הנוכחי (או של האבא שלו).

  • יתרון: גמישות מקומית. אם נגדיל את הפונט של כפתור, הריווחים ב-em יגדלו איתו אוטומטית.
  • שימוש נפוץ: רכיבים מבודדים (Components) שצריכים לשמור על פרופורציות פנימיות.

נקודות שבירה סטנדרטיות (Standard Breakpoints)

למרות שהכי נכון לקבוע נקודות שבירה לפי התוכן שלכם, אלו הטווחים המקובלים ביותר בתעשייה (למשל ב-Bootstrap או Tailwind):

  • 480px (30em): טלפונים גדולים.
  • 768px (48em): טאבלטים (מצב לאורך).
  • 1024px (64em): טאבלטים (מצב לרוחב) ומחשבים ניידים קטנים.
  • 1280px (80em): מסכי מחשב סטנדרטיים.

דוגמה חיה: קופסה עם 3 מצבים

שנו את רוחב הדפדפן כדי לראות את הקופסה משנה צבע וגודל ב-3 נקודות שבירה שונות:

/* Mobile First: Default (Red) */
.box { background: #e74c3c; }

/* Tablet: 768px / 48em (Yellow) */
@media screen and (min-width: 48em) {
  .box { background: #f1c40f; }
}

/* Desktop: 1024px / 64em (Green) */
@media screen and (min-width: 64em) {
  .box { background: #2ecc71; }
}
טיפ: שימוש ב-em בתוך מדיה קווריז (כמו 48em) נחשב ל-Best Practice כי הוא מכבד משתמשים שביצעו זום בדפדפן.
